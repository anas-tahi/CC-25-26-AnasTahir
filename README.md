# CompraSmart 🛒💰

## Hito 1 — Definición del proyecto

**Proyecto:** CompraSmart — Aplicación web de comparación de precios  
**Estudiante:** Anas Tahir  
**Curso:** Cloud Computing: Fundamentos e Infraestructuras — 2025/26  

---

## 📝 Resumen
CompraSmart es una aplicación web desarrollada con **React** (frontend), **Node.js** (backend) y **MongoDB** (base de datos).  
Permite a los usuarios **comparar precios de productos** entre diferentes supermercados de Granada o de toda España.  
La aplicación ayuda a los usuarios a encontrar el **supermercado más barato para cada producto**.  
Finalmente, la aplicación se desplegará en una **plataforma en la nube** para hacerla accesible en línea y escalable.

---

## 🎯 Objetivos
- Ayudar a las personas que llegan por primera vez a Granada o España a encontrar los productos más baratos fácilmente.  
- Permitir a todos los usuarios, especialmente estudiantes, comprar lo que necesitan al **precio más bajo**.  
- Construir una aplicación web funcional utilizando tecnologías modernas (**React, Node.js, MongoDB**).  
- Aprender y aplicar **técnicas de despliegue en la nube** para garantizar accesibilidad y escalabilidad.

---

## 🛠️ Tecnologías Propuestas
- **Frontend:** React.js  
- **Backend:** Node.js con Express.js  
- **Base de Datos:** MongoDB  
- **Despliegue:** Plataforma en la nube (por definir)  
- **Control de Versiones:** Git y GitHub  

---

## 🔧 Arquitectura Inicial
1. Los usuarios interactúan con la aplicación web desde un navegador (**frontend en React**).  
2. El frontend se comunica con el backend (**Node.js + Express**) mediante peticiones API.  
3. El backend obtiene los datos de productos y supermercados desde MongoDB.  
4. La aplicación se desplegará en una plataforma en la nube para garantizar accesibilidad y escalabilidad.

---

## ✅ Requisitos Funcionales (RF)
- Buscar un producto y ver los precios en varios supermercados.  
- Identificar qué supermercado ofrece el **precio más bajo**.  
- Filtrar por **ubicación** o **supermercado** para resultados más precisos.  
- Escalabilidad futura para incluir **más supermercados a nivel nacional**.

---

## ⚡ Requisitos No Funcionales (RNF)
- Diseño responsivo para ordenadores y dispositivos móviles.  
- Tiempos de carga rápidos, incluso con grandes cantidades de datos.  
- Manejo seguro de los datos de los usuarios (para futuras cuentas de usuario).  
- Despliegue en la nube para asegurar disponibilidad y accesibilidad.

# CompraSmart 🛒💰

**Project:** CompraSmart — Price Comparison Web App  
**Student:** Anas Tahir

---

## 📝 Overview
CompraSmart is a web application developed with **React** (frontend), **Node.js** (backend), and **MongoDB** (database).  
It allows users to **compare product prices** across different supermarkets in Granada or throughout Spain.  
The application helps users find the **cheapest supermarket for each product**.  
Eventually, the app will be deployed on a **cloud platform** to make it accessible online and scalable.

---

## 🎯 Objectives
- Help newcomers to Granada or Spain find the cheapest products easily.  
- Allow all users, especially students, to buy what they need at the lowest price.  
- Build a functional web application using modern technologies (**React, Node.js, MongoDB**).  
- Learn and apply **cloud deployment techniques** for accessibility and scalability.

---

## 🛠️ Proposed Technologies
- **Frontend:** React.js  
- **Backend:** Node.js with Express.js  
- **Database:** MongoDB  
- **Deployment:** Cloud platform (TBD)  
- **Version Control:** Git & GitHub  

---

## 🔧 Initial Architecture
1. Users interact with the web application via a browser (**frontend in React**).  
2. The frontend communicates with the backend (**Node.js + Express**) via API requests.  
3. The backend fetches product and supermarket data from MongoDB.  
4. The application will be deployed on a cloud platform for accessibility and scalability.

---

## ✅ Functional Requirements (FR)
- Search for a product and see prices in multiple supermarkets.  
- Identify which supermarket offers the **lowest price**.  
- Filter by **location** or **supermarket** for precise results.  
- Future scalability to include **more supermarkets nationwide**.

---

## ⚡ Non-Functional Requirements (NFR)
- Responsive design for both computers and mobile devices.  
- Fast loading times, even with large datasets.  
- Secure handling of user data (future user accounts).  
- Cloud deployment to ensure availability and accessibility.
